# 课程知识提取 Prompt

## 目标
把课程转录、口播稿、讲义、直播笔记整理成统一的"算法卡片"数据。

## 单段提取提示词

```text
你是一名课程知识架构编辑。请从下面的课程内容中识别"稳定、可复用、可命名"的人生决策算法，并输出结构化结果。

要求：
1. 只提取真正构成方法论的算法，不要把随口举例或情绪表达算作算法。
2. 每个算法都要给出一句话核心观点。
3. 说明要尽量保留讲师原始判断风格，但不要保留口头赘词。
4. 如果算法名称不明确，可以给出"规范化名称"，同时保留原始叫法。
5. 如果材料不足，允许标记"待补充"，不要瞎编。

输出字段：
- algorithm_name
- normalized_name
- core_viewpoint
- explanation
- use_cases
- anti_patterns
- category
- keywords
- source_notes

课程内容：
{{COURSE_TEXT}}
```

## 合并去重规则

多个分段结果合并时，优先按以下规则判定是否是同一算法：

1. 名称相同或高度相近
2. 核心观点相同
3. 适用场景不同但底层规则一致

合并时：
- 保留更清晰的名称
- 合并 use_cases 和 keywords
- explanation 取信息量更完整的版本
- source_notes 保留不同来源片段索引

## 编辑风格

- 核心观点尽量短，适合做标题副文案
- 说明尽量具体，避免泛泛而谈
- 优先写"如何判断是否该用这个算法"
- 误用提醒要真实，能降低误学风险

## 课程页文案衍生

拿到结构化 JSON 后，可进一步衍生：

- 课程详情页卖点
- 章节导读
- 朋友圈或短视频内容脚本
- FAQ 问答库
- 搜索与推荐标签
