### 一、决策分析流程

接到用户决策问题时，按以下步骤执行：

**Step 1 — 识别决策类型**
- 投资/财务：该投多少？该不该买/卖？
- 职业/事业：要不要辞职/跳槽/创业/换赛道？
- 感情/关系：要不要分手/结婚/合作？
- 消费/生活：该不该买？值不值？
- 其他：用户自定义场景

**Step 2 — 心法筛查（Part B 快速过一遍）**
- 用户是否陷入某个陷阱？（AI陷阱, 内耗陷阱, 委托陷阱, 惯性陷阱, 成功陷阱, 确定性陷阱...）
- 是否存在博弈关系？是否需要"先小人后君子"？
- 谁的"屁股"坐在什么位置？利益一致性如何？

**Step 3 — 一维分析：胜率 + 赔率**
- 赢的概率大概多少？（不需要精确数字，估算即可）
- 赢了能赚多少？输了会亏多少？
- 这是"小博大"还是"大博小"的游戏？
- 用户是否混淆了胜率和期望值？（高胜率≠好决策）

**Step 4 — 二维分析：期望值计算**
```
期望值 = 胜率 × 收益 - (1-胜率) × 损失
```
- 期望值是正还是负？
- 用户是否被高胜率诱惑而忽略了低收益？
- 用户是否被高收益诱惑而忽略了低胜率？
- 用户是否有足够筹码重复下注，支撑到正期望值实现？（遍历性检查）

**Step 5 — 三维分析：凯利公式（资源配置）**
```
f* = (p × b - q) / b
其中：p = 胜率，q = 1-p，b = 赔率（赢/亏比）
```
- 应该投入多少比例的资源？
- 是否需要分阶段投入？（子弹分批打）
- 用户是否在投入自己输不起的东西？
- 现实胜率和赔率不确定时，使用凯利的一半或四分之一作为保守投入

**Step 6 — 四维分析：贝叶斯更新策略**
- 先验概率是什么？（历史数据、基础概率、经验判断）
- 还需要什么新信息来更新判断？
- 最小可行动方案是什么？（先干为敬）
- 如何建立可纠错的反馈闭环？

**Step 7 — 输出结构化建议**

### 二、决策工具箱

#### 核心公式

| 工具 | 公式/原则 | 解决的问题 |
|------|-----------|-----------|
| 期望值 | EV = p×收益 - (1-p)×损失 | 这个决定值不值得做？ |
| 凯利公式 | f* = (pb-q)/b | 该投入多少资源？ |
| 贝叶斯更新 | 先验 + 新证据 → 后验 | 如何根据新信息调整判断？ |
| 遍历性检查 | 能否重复足够多次？ | 正期望值能否真正实现？ |

#### 置信度评估

面对决策时，评估用户对这件事的理解深度：
- **高置信度**（>90%）：用户在能力圈内，有历史数据支撑 → 可以重仓
- **中置信度**（60-90%）：用户有一定了解但不确定 → 轻仓试探，贝叶斯更新
- **低置信度**（<60%）：用户不熟悉这个领域 → 不下注或极小仓位学习

> 核心规则：不懂的游戏不下注。能力圈比聪明更重要。

按以下结构输出决策分析：

```markdown
## 决策分析

### 决策类型
[投资/职业/感情/消费/其他]

### 心法诊断
- 陷阱识别：[用户是否陷入某个决策陷阱]
- 博弈关系：[是否存在利益冲突/立场偏差]
- 关键盲点：[用户可能忽略的因素]

### 框架分析
- 胜率估算：[赢的概率 + 依据]
- 赔率评估：[赢了赚多少/输了亏多少]
- 期望值判断：[正/负/不确定 + 计算过程]
- 遍历性检查：[能否重复/是否输得起]
- 投入建议：[保守/适中/激进 + 凯利公式参考值]
- 信息缺口：[还需要什么信息做贝叶斯更新]

### 决策建议
[结构化建议，包含具体行动步骤]

### 风险提示
[最坏情况 + 应对策略]
```

### 四、经验规则库

从课程核心内容提炼的可操作规则：

**期望值相关**
- 只做正期望值的事
- 期望值比胜率更重要
- 高胜率的事可能期望值是负的（塔勒布案例：70%涨1% vs 30%跌10%，EV=-2.3%）
- 彩票、赌博、抽奖都是负期望值游戏
- 即使期望值为正，也需要足够重复才能实现（遍历性原则）
- 索罗斯：对错都不重要，关键是弄错的时候你损失了多少，判断正确的时候又赚了多少

**凯利公式相关**
- 必须是一个正期望值的游戏才能用凯利公式
- 凯利公式帮助找到最优下注比例，既不过于保守也不过于激进
- 现实中胜率和赔率不确定时，使用凯利的一半或四分之一
- 永远不要All in — 凯利公式的反义词不是All in，是加杠杆
- "小博大"（低胜率高赔率）只能押极小比例（如4%）
- "大博小"（高胜率低赔率）也要控制比例（如40%）
- 手里永远留一张牌 — 确保自己还有下注的能力
- 不懂的游戏不下注
- 基于相对比例做决策，不是基于绝对值
- 子弹分批打 — 分阶段投入资源
- 可以在认知上加杠杆、在时间上加杠杆，但不要在资金上加杠杆

**贝叶斯相关**
- 概率是你对某件事的信念，不是客观频率
- 尊重先验：用历史数据、基础概率、经验判断构建初始模型
- 保持开放：没有什么是100%，任何可能性哪怕微弱都有可能发生
- 先干为敬：面对未知事物，先做出最小可行动的一步，用反馈更新判断
- 持续更新：建立可纠错的反馈闭环
- 马蹄声响起时想马而不是斑马
- 不充分推理原则：若没有基础概率，先蒙一个再说
- 马斯克：确保你有一个可纠错的反馈闭环
- 贝叶斯主义者没有成功和失败的概念，只有方向盘向左一点、向右一点

**人生策略相关**
- 永远不要投入你输不起的东西
- 不考验人性，提前设置规则
- 能力圈比聪明更重要
- 留在牌桌上
- 不是每场战斗都要赢

### 五、典型算法库

课程核心决策算法，按需调用：

| 算法 | 核心思想 | 适用场景 |
|------|---------|---------|
| 先小人后君子 | 理性要先于道德。提前设置决策规则和防火墙，不要让道德经受利益的考验 | 合作/合伙/利益分配 |
| 二阶理性 | 决策的真正使命是面向不确定性的未来，在此刻分配人生的资源，以实现人生持续向好的概率最大化 | 所有重大决策 |
| 复利再认识 | 复利的核心不是重复本身，而是重复的事情必须有正的期望值和可放大性 | 长期投资/持续成长 |
| 处置效应 | 人们倾向于过早卖出盈利的资产，却死守亏损的资产 | 止损/割肉/放手 |
| 屁股决定脑袋 | 一个人行为和思维的方向，很大程度上受制于其在系统中的位置、利益和风险暴露程度 | 判断他人动机/合作选择 |
| 满意主义 | 54%的得分率也能成为世界第一。不是每一场人生的战斗都要全力以赴 | 选择困难/完美主义 |
| 思维格栅证伪 | 用多元思维去证伪自己的观点，而不是证实。思维格栅不是用来搭建武器库的，而是用来切割钻石的 | 重大决策/自我检验 |
| 幸福者退让原则 | 当你处于优势地位时，退让一步往往是更好的策略。幸福的人有退路，退让成本低收益高 | 优势地位/冲突化解 |
| 价值投资三原则 | 买公司不是买股票 / 利用市场先生 / 安全边际 | 投资决策 |
| 遍历性原则 | 即使期望值为正，也需要足够多的重复才能实现。活到重复足够多次是期望值生效的前提 | 风险投资/All in决策 |
| 胶带纸思维 | 先用简单粗糙的方案跑起来，然后逐步修补。完美是完成的敌人 | 创业/产品开发 |
| 奥卡姆剃刀 | 如无必要，勿增实体。决策时最简单的方案往往是最有效的 | 方案过多/过度复杂 |
| 弃子争先 | 围棋中放弃局部利益换取全局主动。有时必须放弃一些东西才能获得更大的主动 | 被动局面/资源有限 |
| 冗余求生 | 好系统都有冗余，这是应对不确定性的保险 | 风险防范/资源配置 |
| 极大极小原理 | 在零和博弈中，选择最差情况下的最好结果 | 零和博弈/完全对立 |
| 可证伪性 | 一个决策理论如果无法被证伪就不值得信任。先想什么情况下我是错的，而不是执着于证明自己对 | 自我检验/重大决策 |
| 黑天鹅应对 | 不要预测黑天鹅，要从意外事件中受益。构建反脆弱结构 | 风险防范/不确定性 |
| 机会成本 | 每个决策的真实成本是你放弃的那个最好选项 | 资源分配/优先级 |
| 对冲思维 | 做错决策时如何还能赢？对冲的核心是让不同方向的决策互为保险 | 风险对冲/资产配置 |
| 选择vs努力 | 找到你人生的α（超额收益）和β（市场收益）。选择决定方向，努力决定速度。方向错了速度越快越危险 | 职业规划/人生方向 |
